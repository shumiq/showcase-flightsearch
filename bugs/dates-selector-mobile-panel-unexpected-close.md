# Dates Selector Mobile Panel Closes on Any Interaction

**Status:** Open  
**Severity:** High  
**Date Created:** 2026-05-06  

---

## Description
On mobile viewports, the DatesSelector panel in the FlightSearch component closes immediately when the user taps anywhere inside the panel — including the calendar content area and empty spaces. This makes it impossible to select dates, change trip types, or interact with any element within the mobile calendar modal. The expected overlay backdrop tap-to-close behavior works correctly, but all taps inside the panel content incorrectly trigger the same close logic.

## Steps to Reproduce
1. Open the FlightSearch component on a mobile viewport (screen width < 768px)
2. Tap the date input field to open the DatesSelector panel
3. Tap anywhere inside the opened panel (calendar dates, empty areas, trip type radio buttons, or any other element)
4. Observe that the panel closes immediately instead of registering the interaction

**Expected Behavior:**  
Taps inside the panel content area should register normally — selecting dates, toggling trip types, or scrolling the calendar. Only taps on the dark overlay backdrop should close the panel.

**Actual Behavior:**  
Every tap inside the panel (including empty space and interactive elements) causes the panel to close immediately, preventing any interaction with the calendar.
