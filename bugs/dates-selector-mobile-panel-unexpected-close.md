# Dates Selector Mobile Panel Closes on Any Interaction

**Status:** Open  
**Severity:** High  
**Date Created:** 2026-05-06  

---

## Description
On mobile viewports, the DatesSelector panel in the FlightSearch component closes immediately when the user taps anywhere inside the panel — including the calendar content area and empty spaces. This makes it impossible to select dates, change trip types, or interact with any element within the mobile calendar modal. The expected overlay backdrop tap-to-close behavior works correctly, but all taps inside the panel content incorrectly trigger the same close logic.

## Steps to Reproduce
1. Open the FlightSearch component on a mobile viewport (screen width < 768px)
2. Tap the date input field to open the DatesSelector panel
3. Tap anywhere inside the opened panel (calendar dates, empty areas, trip type radio buttons, or any other element)
4. Observe that the panel closes immediately instead of registering the interaction

**Expected Behavior:**  
Taps inside the panel content area should register normally — selecting dates, toggling trip types, or scrolling the calendar. Only taps on the dark overlay backdrop should close the panel.

**Actual Behavior:**  
Every tap inside the panel (including empty space and interactive elements) causes the panel to close immediately, preventing any interaction with the calendar.

## Investigation Findings & Technical Context
*(Generated by AI Code Search)*

- **Suspected Files / Components:** `src/components/FlightSearch/DatesSelector.tsx`
- **Technical Clues:** 
  - Lines 146-147: `dropdownRef` and `triggerRef` are defined, but `dropdownRef` is only attached to the **desktop calendar** (`data-testid="desktop-calendar"`, line 461), which has the class `hidden md:block` — meaning it is hidden on mobile.
  - Lines 159-174: A `mousedown` event listener is registered on `document` when `isOpen` is true. The `handleClickOutside` function checks if the click target is outside both `dropdownRef` and `triggerRef`. If both checks pass, it calls `setIsOpen(false)`.
  - Lines 466-488: The mobile calendar modal (`data-testid="mobile-calendar"`) has **no ref** attached to its container.
  - On mobile, when the user taps anywhere inside the mobile calendar, `mousedown` fires. Since the tapped element is neither inside the hidden desktop `dropdownRef` nor the `triggerRef`, both `contains` checks fail, and the panel closes.
- **Potential Root Cause:** The click-outside handler does not account for the mobile panel. The `dropdownRef` only references the desktop calendar (hidden on mobile), so every tap on mobile is incorrectly treated as a click-outside.
- **Missing Information / Gaps:** None — root cause is clear from code inspection.
